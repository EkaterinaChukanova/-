    Семантика выполнения Poof (runtime)

Документ относится только к интерпретации уже построенного AST: порядок вычислений, области видимости и ошибки времени выполнения. Конкретный синтаксис и построение AST — зона парсера/лексера.

    Модель вычислений

- Вызов по значению (call-by-value). Сначала вычисляются подвыражения, затем применяется операция или вызов.
- Результат вычисления — значение типа `Value` (см. `Environment.fs`).
- Интерпретатор раскручивает шаги вычисления через `Trampoline` (см. `Interpreter.fs`): хвостовые переходы к следующему выражению оформляются как `Step (fun () -> …)` и снимаются в цикле `runTramp`, без неограниченного роста стека вызовов F# для хвостовых программ на Poof.

    Окружение

- Переменные хранятся в цепочке слоёв: текущая карта `Bindings` и опциональный `Parent`.
- Поиск (`lookup`): сначала текущий слой, затем родительские.
- Связывание через `bindValue`: имя добавляется в текущий слой. Для рекурсии имена могут быть связаны с обновляемой ячейкой (`VRef`).

    Выражения (кратко)

| Конструкция AST | Поведение |
|-----------------|-----------|
| Литералы | Значение как есть. |
| `Var` | Поиск в окружении; при отсутствии — ошибка. |
| `BinOp` | Сначала левое и правое поддеревья, затем операция (часть операций требует конкретных типов). |
| `UnariMinus` | Одно число (int или float). |
| `If` | Условие → bool; затем ровно одна из веток. |
| `Lambda` | `VFunc(param, body, env)` захватывает окружение на момент создания. |
| `Apply` | Вычислить функцию и аргумент; затем `VFunc` или `VBuiltin`. |
| `Bind` | Имя доступно во время вычисления правой части; затем тело. |
| `Cast` / `Invoke` | См. код интерпретатора (вызов по значению, цепочка для `invoke`). |
| `When` | Вычислить subject, затем ветки по порядку до первого совпадения паттерна. |
| `Sequence` | Подвыражения слева направо, результат — последнее значение. |

    Хвостовая рекурсия (TCO, бонус)

- Интерпретатор не переходит по хвостовым вызовам через глубокую рекурсию F# между `eval` и вложенными вызовами: следующий шаг возвращается как `Step` и обрабатывается в `runTramp`.
- Не-хвостовая рекурсия (например, умножение после рекурсивного вызова) по-прежнему может требовать стека под вычисление подвыражений.

    Ленивость (бонус)

- Ритуал `delay` принимает функцию `f`, применённую затем к `()` внутри `Lazy<Value>` (в Poof: одноарность; для « thunk » обычно пишут `spell t _ -> …` и подают `cast t`).
- Ритуал `force` вычисляет и мемоизирует результат (поведение `Lazy` в .NET).
- Обычный вызов функции и операторы остаются строгими по аргументам.

    Ошибки времени выполнения

Используется `RuntimeError` и функции `require*` там, где тип значения не подходит.

    Граница ответственности

- Поведение `Poof.Interpreter` и встроенных ритуалов в `initGlobalEnv`: семантика вычислений и ошибок здесь.
